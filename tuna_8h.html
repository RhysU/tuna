Detailed Description

Tuna public API. Designed to have the minimial possible set of dependencies. The implementation is not thread-safe but it can be used in thread-safe fashion provided that some external lock guards access to struct instances.

◆ tuna_stats

struct tuna_stats

Provides statistical accumulators for special cases of interest.

Accumulates running mean and variance details from a data stream. Fill storage with zeros, e.g. from POD zero initialization, to construct or reset an instance.

Adapted from Suzerain's suzerain::running_statistics class which adapted it from http://www.johndcook.com/standard_deviation.html. Rewritten in C and extended to permit merging statistics from multiple instances. Storage overhead reduced relative to Cook's presentation.

◆ tuna_algo_default()

const tuna_algo* tuna_algo_default ( const size_t  nchunk)

Retrieve a default algorithm when one is left unspecified.

If nchunk < 2, the "zero" algorithm is returned. If the whitespace-trimmed environment variable TUNA_ALGO case-insensitively names an algorithm (e.g., "welch1", "welch1_nuinf", "zero"), that algorithm will be used. Otherwise, a sensible default which may or may not take into account nchunk is chosen.

Human-readable output for summarizing runtime behavior.

The simpler, non-varargs *_fprint routines are intended for non-C purposes while the more flexible but less typesafe *_fprintf variants are intended for C usage. The stream parameters to these methods are not flushed. Output nchunk+1 status line(s) about tuna_site site and associated tuna_chunk chunks[0], ..., chunks[nchunk-1] to FILE* stream prefixed by prefix.

Parameters
streamFILE* on which output is produced.
siteSite for which information is output.
chunksChunks about which information is output.
nchunkNumber of contiguous chunks in chunks.
prefixA string used to prefix the output.
labelsIf non-NULL and non-trivial, labels[ik] labels the ikth chunk in the output. Otherwise, each chunk is labelled with its index ik.
Returns
The number of characters output on success. On error, a negative value is returned.
